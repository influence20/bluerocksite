SSL Certificate Implementation Guide

Comprehensive instructions for implementing SSL certificates to secure your BlueRock Asset Management website

Understanding SSL Certificates

SSL (Secure Sockets Layer) certificates are digital certificates that authenticate the identity of a website and enable an encrypted connection. These certificates are essential for securing sensitive information transmitted between your website and its visitors.

Enhanced Security

SSL certificates encrypt data transmission, protecting sensitive information like login credentials, personal details, and financial data from interception by malicious actors.

Trust & Credibility

Websites with SSL certificates display a padlock icon in the browser address bar, signaling to visitors that their connection is secure and building trust in your brand.

SEO Advantage

Google and other search engines prioritize secure websites in search results, giving SSL-enabled sites a ranking advantage over non-secure alternatives.

Types of SSL Certificates

There are several types of SSL certificates available, each offering different levels of validation and security features:

Certificate Type Validation Level Features Best For
Domain Validated (DV) Low Basic encryption, quick issuance (minutes to hours) Blogs, personal websites, small businesses
Organization Validated (OV) Medium Enhanced encryption, organization verification (1-3 days) Business websites, e-commerce sites
Extended Validation (EV) High Maximum encryption, rigorous verification (1-2 weeks) Financial institutions, large e-commerce sites
Wildcard SSL Varies Secures main domain and unlimited subdomains Websites with multiple subdomains
Multi-Domain SSL (SAN) Varies Secures multiple domains with one certificate Organizations with multiple websites

Recommendation: For BlueRock Asset Management, we recommend using an Organization Validated (OV) or Extended Validation (EV) certificate to provide the highest level of security and trust for your financial services website.

SSL Implementation by Hosting Provider

The process of implementing an SSL certificate varies depending on your hosting provider. Below are step-by-step instructions for popular hosting platforms:

SSL Implementation in cPanel

  1. Log in to cPanel

    Access your hosting account's cPanel dashboard using your credentials.

  2. Navigate to SSL/TLS Manager

    In the Security section, click on "SSL/TLS Manager" or "SSL/TLS".

  3. Select Certificate Installation

    Click on "Install and Manage SSL for your site (HTTPS)" or a similar option.

  4. Choose Domain

    Select the domain for which you want to install the SSL certificate.

  5. Install Certificate

    You have two options:

    • Auto SSL: Many hosts offer free Let's Encrypt certificates. Click "Run AutoSSL" if available.
    • Manual Installation: If you purchased a certificate from a provider, enter the certificate details (Certificate, Private Key, and CA Bundle) in the appropriate fields.
  6. Install and Verify

    Click "Install Certificate" and wait for the installation to complete. Then visit your website using HTTPS to verify the certificate is working properly.

  7. Set Up Redirects (Optional but Recommended)

    To force all traffic to use HTTPS, create a redirect rule in the .htaccess file:

    RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

SSL Implementation in Plesk

  1. Log in to Plesk

    Access your Plesk control panel using your administrator credentials.

  2. Select Your Domain

    From the list of domains, click on the domain you want to secure with SSL.

  3. Access SSL/TLS Certificates

    In the domain management page, find and click on "SSL/TLS Certificates" under the Security section.

  4. Add SSL Certificate

    Click on "Add SSL Certificate" or "Secure Your Site" button.

  5. Choose Certificate Source

    You have several options:

    • Let's Encrypt: Click "Get Free Let's Encrypt Certificate" for a free, auto-renewing certificate.
    • Buy a Certificate: Purchase a certificate through Plesk's interface.
    • Use Custom Certificate: Upload your own certificate files if you already have them.
  6. Configure Certificate Details

    Fill in the required information, including domain names to be covered by the certificate.

  7. Install and Activate

    Complete the installation process and ensure the certificate is activated for your domain.

  8. Enable HTTPS Redirection

    In the domain's settings, look for "HTTPS Settings" or "Permanent SEO-Safe 301 Redirect" and enable redirection from HTTP to HTTPS.

SSL Implementation in AWS

  1. Access AWS Certificate Manager (ACM)

    Log in to your AWS Management Console and navigate to Certificate Manager.

  2. Request a Certificate

    Click "Request a certificate" and select "Request a public certificate".

  3. Enter Domain Names

    Add your domain name(s). You can include both the root domain (example.com) and subdomains (www.example.com).

  4. Choose Validation Method

    Select either DNS validation (recommended) or email validation.

    • DNS Validation: You'll need to create CNAME records in your domain's DNS settings.
    • Email Validation: AWS will send verification emails to domain-associated addresses.
  5. Complete Validation

    Follow the validation steps provided by AWS to prove domain ownership.

  6. Configure AWS Service

    Once the certificate is issued, you need to configure it with your AWS service:

    • For CloudFront: In your distribution settings, select the certificate under "SSL Certificate".
    • For Elastic Load Balancer: In the listener configuration, add an HTTPS listener and select your certificate.
    • For API Gateway: In the custom domain settings, select your certificate.
  7. Update DNS Records

    Ensure your domain's DNS records point to your AWS resources (e.g., CloudFront distribution or load balancer).

  8. Test HTTPS Connection

    Visit your website using HTTPS to verify the certificate is working correctly.

Tip: AWS Certificate Manager provides free SSL/TLS certificates that automatically renew, but they can only be used with AWS services like CloudFront, API Gateway, and Elastic Load Balancer.

SSL Implementation in Microsoft Azure

  1. Log in to Azure Portal

    Access your Azure Portal at portal.azure.com.

  2. Navigate to App Service

    Find and select your App Service or Web App from the resources list.

  3. Access TLS/SSL Settings

    In the left menu, under Settings, click on "TLS/SSL settings".

  4. Add Certificate

    You have several options:

    • Create App Service Managed Certificate: Free certificate for custom domains (not available for root domains).
    • Import App Service Certificate: Use a certificate purchased through Azure.
    • Upload Certificate: Import your own PFX certificate file.
    • Import from Key Vault: Use a certificate stored in Azure Key Vault.
  5. Bind Certificate to Domain

    After adding the certificate, go to "TLS/SSL bindings" and click "Add TLS/SSL binding".

  6. Configure Binding

    Select your custom domain, the certificate, and the TLS/SSL type (SNI SSL or IP-based SSL).

  7. Add Binding

    Click "Add Binding" to complete the process.

  8. Configure HTTPS Only

    For security, enable "HTTPS Only" in your app's TLS/SSL settings.

Note: Azure App Service Managed Certificates are free but have limitations. For production environments with high security requirements, consider using a purchased certificate or importing a certificate from a trusted provider.

SSL Implementation with Let's Encrypt (Manual)

Let's Encrypt is a free, automated, and open certificate authority that provides SSL certificates. Here's how to implement it manually on your server:

  1. Install Certbot

    Certbot is the recommended client for Let's Encrypt. Install it on your server:

    # For Ubuntu/Debian
sudo apt-get update
sudo apt-get install certbot

# For CentOS/RHEL
sudo yum install certbot
  2. Obtain Certificate

    Run Certbot to obtain a certificate. The command varies based on your web server:

    # For Apache
sudo certbot --apache -d example.com -d www.example.com

# For Nginx
sudo certbot --nginx -d example.com -d www.example.com

# Standalone (if you don't have Apache or Nginx)
sudo certbot certonly --standalone -d example.com -d www.example.com
  3. Verify Certificate Installation

    Visit your website using HTTPS to ensure the certificate is working correctly.

  4. Set Up Auto-Renewal

    Let's Encrypt certificates expire after 90 days. Set up automatic renewal:

    # Test automatic renewal
sudo certbot renew --dry-run

# Add cron job for automatic renewal
echo "0 0,12 * * * root python -c 'import random; import time; time.sleep(random.random() * 3600)' && certbot renew -q" | sudo tee -a /etc/crontab > /dev/null
  5. Configure HTTPS Redirection

    For Apache, add to your .htaccess file:

    RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

    For Nginx, add to your server block:

    server {
    listen 80;
    server_name example.com www.example.com;
    return 301 https://$host$request_uri;
}

Warning: Manual installation requires server access and technical knowledge. If you're not comfortable with server administration, consider using a hosting provider that offers Let's Encrypt integration through their control panel.

SSL Configuration Best Practices

Follow these best practices to ensure optimal security and performance with your SSL implementation:

  1. Use Strong SSL/TLS Protocols

    Disable outdated protocols (SSL 2.0, SSL 3.0, TLS 1.0, TLS 1.1) and only enable TLS 1.2 and TLS 1.3.

    # Apache configuration
SSLProtocol -all +TLSv1.2 +TLSv1.3

# Nginx configuration
ssl_protocols TLSv1.2 TLSv1.3;
  2. Configure Strong Cipher Suites

    Use strong, modern cipher suites and disable weak ones:

    # Apache configuration
SSLCipherSuite EECDH+AESGCM:EDH+AESGCM:AES256+EECDH:AES256+EDH

# Nginx configuration
ssl_ciphers 'EECDH+AESGCM:EDH+AESGCM:AES256+EECDH:AES256+EDH';
  3. Enable HTTP Strict Transport Security (HSTS)

    HSTS tells browsers to only use HTTPS for your domain:

    # Apache configuration
Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"

# Nginx configuration
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
  4. Implement Proper Redirects

    Ensure all HTTP traffic is redirected to HTTPS using 301 redirects.

  5. Update Mixed Content

    Ensure all resources (images, scripts, stylesheets) are loaded over HTTPS to avoid mixed content warnings.

  6. Set Secure Cookies

    Add the Secure and HttpOnly flags to cookies:

    # PHP example
session_set_cookie_params([
    'secure' => true,
    'httponly' => true,
    'samesite' => 'Lax'
]);
  7. Implement Content Security Policy (CSP)

    Add a CSP header to protect against XSS attacks:

    # Apache configuration
Header set Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.google-analytics.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: https://www.google-analytics.com; connect-src 'self' https://www.google-analytics.com; font-src 'self'; frame-src 'self'; object-src 'none'"

# Nginx configuration
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.google-analytics.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: https://www.google-analytics.com; connect-src 'self' https://www.google-analytics.com; font-src 'self'; frame-src 'self'; object-src 'none'";
  8. Regular Certificate Monitoring

    Set up monitoring to alert you before certificates expire. Tools like Certbot can handle automatic renewal for Let's Encrypt certificates.

  9. Test SSL Configuration

    Use tools like SSL Labs to test your SSL implementation and identify potential issues.

Pro Tip: After implementing SSL, use the Qualys SSL Server Test to verify your configuration. Aim for an A+ rating by following the best practices above.

Troubleshooting Common SSL Issues

Here are solutions to common SSL implementation issues:

Mixed Content Warnings

Problem: Browser shows "Mixed Content" warnings when some resources are loaded over HTTP instead of HTTPS.

Solution:

  1. Identify mixed content using browser developer tools (F12 > Console).
  2. Update all resource URLs to use HTTPS instead of HTTP.
  3. For dynamic content, use protocol-relative URLs (//example.com/resource) or absolute paths (/resource).
  4. If you can't modify third-party content, consider using a Content Security Policy or a proxy.

Certificate Not Trusted

Problem: Browsers show "Your connection is not private" or "Certificate not trusted" errors.

Solution:

  1. Ensure your certificate is issued by a trusted Certificate Authority.
  2. Check if the intermediate certificates are properly installed.
  3. Verify that your certificate matches the domain name being accessed.
  4. Check certificate expiration date.

Certificate Name Mismatch

Problem: Browser shows "Certificate name mismatch" error.

Solution:

  1. Ensure your certificate covers the exact domain being accessed (www.example.com vs example.com).
  2. For multiple domains or subdomains, use a multi-domain or wildcard certificate.
  3. Set up proper redirects to ensure users access the correct domain version.

Performance Issues

Problem: Website loads slowly after implementing SSL.

Solution:

  1. Enable HTTP/2 support, which performs better with SSL.
  2. Implement OCSP stapling to speed up certificate validation.
  3. Use session resumption to reduce handshake overhead.
  4. Enable TLS 1.3 for faster connection establishment.
  5. Optimize your SSL configuration for performance.

Redirect Loops

Problem: Browser shows "Too many redirects" error after implementing HTTPS redirects.

Solution:

  1. Check your redirect rules to ensure they only redirect HTTP to HTTPS, not HTTPS to HTTPS.
  2. Verify that your SSL is properly installed and working.
  3. Clear browser cache and cookies.
  4. Check for conflicts with CDN or proxy settings.
